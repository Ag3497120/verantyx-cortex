# Verantyx JCross v4 (漢字型記法) 完全仕様書

このドキュメントは、Verantyx Engine の根幹である **JCross v4 空間記憶フォーマット** を、誰もが直感的に理解し、システム（LLMや人間）にエラーなく正確に読み書きさせるための公式仕様書です。

## 概要 (What is JCross v4?)

JCrossは、ただのマークダウンやJSONではありません。「概念」をゼロコピーでシステム内にキャッシュレイアウトし、多次元のトポロジー（空間的関係性、時間、重要度、抽象度など）を持ったままシリアライズ／デシリアライズするための **Declarative Semantic Graph** （宣言的意味グラフ）アーキテクチャです。

拡張子: `.jcross`

---

## 📄 フォーマット構造 (基本テンプレート)

各JCrossノードは、厳格に以下のブロック構造に従って記述されます。
**改行やデリミタ（`===` や `---`）もパーサの基準になるため、正確に入力してください。**

```text
■ JCROSS_NODE_{ノードのユニークキーID}

【空間座相】
[漢字タグ:ウェイト] [漢字タグ:ウェイト]

【次元概念】
このノードが表す短い概念名やタイトル

【時間刻印】
YYYY-MM-DDTHH:MM:SS+00:00

【連帯】
{対象ノードID}:{関係性タイプ}:{接続強度(0.0~1.0)}

【抽象度】
<0.0 ~ 1.0>

【環境刻印】
(オプション) 依存するファイルパス、URL、実行環境などのHash

【反射】
(オプション) 自発的アクションキューや実行スクリプトなど

---
[本質記憶]
このノードの具体的な内容。コードブロック、長文テキストなど、
どのようなフォーマットでも許容されるメインのペイロード空間です。
===

```

## 🧮 各項目の詳細解説

### `■ JCROSS_NODE_` (ノード宣言)
必ずファイルの先頭に記述します。`tm_` プリフィックス（自動スナップショット用）や `void_` プリフィックス（知識欠落用）などで始まります。
例: `■ JCROSS_NODE_void_45f0d14034e1`

### `【空間座相】` (Kanji Ontology Tags)
**JCrossの最大の特徴です。** JSONのように細かいプロパティを列挙するのではなく、「漢字1文字」のベクトル修飾子（タグ）を使って、記憶エンジンの重力空間における挙動を決定します。
記法: `[漢字:重み]`　（重みは 0.0 ~ 1.0 またはそれ以上の係数）
例: `[視:0.9] [認:0.8] [庫:1] [深:0.5]`

#### 必須・標準の漢字オントロジー (コア軸)
JCrossエンジンは以下のコア漢字ベクトルを解析し、記憶の減衰率や重力定数を変化させます。

1. **時間軸 (Time)**
   - `[新:1.0]` : 最新の記憶。（重力増、減衰普通）
   - `[古:1.0]` : 古い記憶。（重力減、減衰大）
   - `[瞬:1.0]` : 一時的なキャッシュやログ。（急激に減衰する）
   - `[永:1.0]` : 永遠の真理や法則。（減衰しない）
2. **空間軸 (Space)**
   - `近`, `遠`, `内`, `外`, `深`, `浅`
3. **抽象度 (Abstraction)**
   - `[具:1.0]` : 具体的な実装コードや事実。
   - `[抽:1.0]` : 抽象的な概念やアーキテクチャ。
   - `[元:1.0]` : メタ情報。
   - `[細:1.0]` : 細かなディテール情報。
4. **信頼度・確度 (Confidence)**
   - `[確:1.0]` : 確認済みの事実。
   - `[疑:1.0]` : 疑わしい情報。
   - `[仮:1.0]` : 仮説（Hypothesis）。
   - `[偽:1.0]` : 破棄フラグ。この空間記憶は排除される（Purge Flag）。
5. **重要度 (Importance)**
   - `[重:1.0]` : 非常に重要。
   - `[軽:1.0]` : どうでもいい情報。
   - `[核:1.0]` : システムのコア。絶対保存。
   - `[周:1.0]` : 周辺情報。
6. **指向性・意図 (Intent)**
   - `[探:1.0]` : 不足している情報を探りに行く意図（Epistemic Drive トリガー）。
   - `[攻:1.0]` : 書き換えや積極的変革アクション。
7. **状態 (State)**
   - `[動:1.0]` : 実行可能なアクション、可変状態。
   - `[静:1.0]` : 静的なデータ、不変状態。
   - `[変:1.0]` : 変更対象、修正パッチ。
   - `[固:1.0]` : ロックされた状態、完成品。
8. **技巧・手続き (Procedural Skill)**
   - `[術:1.0]` : **実行可能な手順や思考の型（スキル）**。この空間座標を持つノードは、ファイル等に隣接した時に「解決プロセス」として振る舞い、強力な重力と遅い減衰率を持ちます。

### `【次元概念】`
このノードが一体何についてのグラフ空間なのかを表す文字列。Markdownでいうタイトル(`<h1>`)に相当します。
例: `Zero-Copy Semantic Graph Layout (ZCSGL)`

### `【時間刻印】`
標準的な ISO 8601 フォーマット。
例: `2026-04-11T05:18:05+00:00`

### `【連帯】` (Relationship Links)
グラフノード同士のネットワークを形成します。指定されたIDを持つ他のJCrossファイルと論理的なリンクを張ります。
改行区切りで複数定義可能です。

**記法:** `{Target_Key}:{Relation_Type}:{Strength}`

**Relation_Type (関係クラス) の一覧:**
- `派生` (Derived) : その知識から派生した新しい状態。
- `基底` (Base) : 派生元のオリジナル。
- `類似` (Similar) : 似ている概念や設計。
- `対立` (Opposite) : パラダイムが対立する概念。
- `前項` (Prev) / `次項` (Next) : 履歴やログのフロー。
- `因果` (Cause) : バグや現象の原因と結果。
- `訂正` (Fix) : 古い情報を上書きする訂正ノード機能。
- `補足` (Context) : 暗黙の文脈の参照。

**記述例:**
```text
【連帯】
tm_0d61dd0628a8:基底:0.9
void_45f0d14034e1:因果:0.5
```

### `【抽象度】`
0.0 から 1.0 までの Float 値。（1.0が完全な理論/概念、0.0が完全な物理的ファイル等に該当）

### `【環境刻印】` (オプション)
これがソースコードの実装スナップショットである場合、その対象への相対ファイルパスなどを記録します。存在しない場合は空にして構いません。
例: `./crates/ronin-core/src/memory_bridge/spatial_index.rs`

### `【反射】` (オプション)
システム自身の**「自発的アクションキュー（Epistemic Drive）」**やスクリプトを格納します。
ここにテキストやスクリプトがあると、スウォームオーケストレーターがこれを検知し、自律的にブラウザ検索やLLM推論を起動します。

### `[本質記憶]` 〜 `===` (ペイロード)
`---` から始まり、 `[本質記憶]` というマーカーの直後から巨大なペイロード（知識そのもの、プログラムのソースコード、対話履歴、など）が記述されます。
終了マーカーは必ず `===` を単独行として配置してください。

## 📝 完璧なサンプル

以下は、ある特定のアーキテクチャ欠落（Void）を自発的に探求しようとするノードの出力例です。

```text
■ JCROSS_NODE_void_45f0d14034e1

【空間座相】
[探:0.95] [仮:0.8] [核:0.5]

【次元概念】
Zero-Copy Semantic Graph Layout 解析課題

【時間刻印】
2026-04-11T05:18:00+00:00

【連帯】
tm_abstract_ast_001:基底:0.9

【抽象度】
0.9

【環境刻印】
./architecture/v4_planning.txt

【反射】
- RustにおけるZero-copy AOT compilationの最適化手法をブラウザで検索する
- 抽象構文木の統一処理を設計する

---
[本質記憶]
現在のJCrossレンダリングにおいて、各ノードの相互参照時にメモリアロケーションが大量に発生している。
このエントロピーを解消するため、非直列化による直接メモリマッピング技術の概念(Void)を埋める必要がある。
===

```

このフォーマットを守れば、どんな人間や別モデル（Ollama, Qwenなど）であっても、一発で Verantyx Spatial Index に安全に知識を inject できるようになります。
